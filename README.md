# zh-portal-resource

## 项目简介

zh-portal-resource 是一个用于归档和发布多个独立 HTML 页面的仓库。这些页面不针对任何特定域名或网站，而是作为通用资源进行维护。仓库的主要目的是方便页面资源的集中管理与版本追踪。

## 目录说明

```
zh-portal-resource/
├── pages/          # 存放所有独立 HTML 页面文件
├── assets/         # 页面引用的静态资源（CSS、JavaScript、图片等）
└── README.md       # 本文件
```

- **pages/**：每个 HTML 页面以独立文件形式存放，文件名应能反映页面用途。
- **assets/**：公共资源文件，页面可通过相对路径引用。

## 页面归档说明

- 所有页面均为独立的 HTML 文件，不依赖外部服务或特定域名。
- 页面内容保持简洁，避免嵌入过多外部资源，以降低维护成本。
- 归档的页面可能来自不同时期或不同用途，但均遵循相同的基本规范。

## 维护说明

- 提交新页面时，请将 HTML 文件放入 `pages/` 目录，并确保其引用的资源位于 `assets/` 下。
- 如需更新已有页面，建议保留旧版本或通过 Git 历史记录回溯。
- 不建议在页面中包含任何敏感信息、动态脚本或对外部服务的强依赖。
- 本仓库不保证对页面内容进行定期更新，仅作为存档和发布用途。

本仓库内容按原样提供，不附带任何明示或暗示的保证。使用者可根据需要自由使用、修改和分发，但需自行承担相关责任。
